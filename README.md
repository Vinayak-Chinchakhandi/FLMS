# 🎓 IFLO — Intelligent Faculty Leave Orchestrator

A smart system designed to automate faculty leave management, optimize class substitutions, and ensure uninterrupted academic operations.

---

## 🧠 The Idea

In most institutions, faculty leave handling is manual, inefficient, and often leads to:

* Missed or poorly managed classes
* Uneven workload distribution
* Last-minute substitution chaos

**IFLO solves this by introducing intelligence into the workflow.**

It transforms a simple leave request into a fully managed process involving:

* Impact evaluation
* Smart substitution planning
* Role-based approvals
* Continuity assurance

---

## 🚀 What Makes IFLO Different

Instead of just managing leave requests, IFLO focuses on:

### ⚡ Academic Continuity

Every leave is evaluated based on how it affects scheduled classes.

### 🤖 Smart Substitution

The system suggests the best possible substitute faculty based on:

* Subject expertise
* Availability
* Workload balance

### 📊 Impact Awareness

Before approval, the system calculates:

* Number of affected classes
* Substitution feasibility
* Overall disruption score

### 🔁 Dynamic Role Handling

When a Head of Department (HOD) is on leave:

* An **Acting HOD** is automatically assigned
* That user gains temporary elevated access
* Ensures decision-making never stops

---

## 🏗️ Core Features

### 👨‍🏫 Faculty Module

* Apply for leave with date range and reason
* Track leave usage and remaining balance
* View assigned substitutions

### 👩‍💼 HOD Module

* Approve or reject leave requests
* Analyze impact before approval
* Trigger substitution simulation

### 🔄 Substitution Workflow

* Automatic substitute suggestions
* Accept/reject mechanism for assigned faculty
* Maintains fairness in workload

### 🧑‍💼 Acting HOD System

* Auto-triggered during HOD absence
* Temporary authority transfer
* Dual access (faculty + HOD capabilities)

---

## 🗄️ System Design

The system is built around structured data relationships:

* **Users** → Faculty & HOD roles
* **Departments** → Organizational structure
* **Timetable** → Class schedules
* **Leave Requests** → Core workflow driver
* **Substitutions** → Class reassignment logic
* **Impact Logs** → Decision intelligence
* **Acting HOD Assignments** → Role continuity
